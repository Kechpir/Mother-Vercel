# Проект: Session Automation System (Zoom + Telegram + Prodamus)

## 1. Общее описание
Автоматизированная система продажи и проведения сессий. 
Клиент оплачивает через Prodamus на лендинге, получает доступ в Telegram-бот, который выдает одноразовую ссылку на закрытую группу. В группе ежедневно публикуются ссылки на Zoom-сессии.

## 2. Технологический стек
- **Frontend/Backend:** Next.js (Vercel) — лендинг, дашборд и API для оплат.
- **Database:** Supabase (PostgreSQL) — хранение данных участников и статусов оплат.
- **Payment:** Prodamus (интеграция через Webhooks).
- **Bot:** Node.js (Telegraf) — Telegram-бот.
- **Meeting:** Zoom (управление сессиями).
- **AI:** OpenAI API (техподдержка в ТГ-боте).

## 3. Ключевые функции

### А. Лендинг и Оплата (Приоритет №1)
- Форма: ФИО, Телефон, Город, Почта.
- Интеграция с Prodamus.
- Страница успеха с переходом в ТГ-бот.
- Наличие оферты и контактов для прохождения модерации платежки.

### Б. Telegram Бот и Группа
- Проверка оплаты через БД Supabase.
- Генерация и выдача одноразовой ссылки на вступление в закрытую группу.
- Ежедневный автоматический постинг ссылки на Zoom в группу ("Ссылка на сеанс").
- Защита контента в группе (запрет копирования/пересылки).

### В. AI-Техподдержка
- Бот в личных сообщениях отвечает на вопросы пользователей по продукту.
- Возможность переключения на живого оператора.

### Г. Админ-Дашборд
- Мониторинг записей и оплат (графики, таблицы).
- Управление базой участников.

## 4. План реализации

### Этап 1: Лендинг и Платежи (Текущий этап)
1. Анализ референса лендинга.
2. Подготовка структуры Next.js + Supabase.
3. Настройка формы регистрации и связи с БД.
4. Настройка Webhook от Prodamus для автоматического подтверждения оплаты.

### Этап 2: Telegram Бот
1. Создание бота и логики "проверки билета".
2. Реализация выдачи инвайт-ссылок.
3. Настройка ежедневного планировщика сообщений (Cron).

### Этап 3: AI и Поддержка
1. Интеграция OpenAI в бота.
2. Настройка промпта и логики перевода на оператора.

### Этап 4: Дашборд и Полировка
1. Создание админ-панели.
2. Финальное тестирование всей воронки.
